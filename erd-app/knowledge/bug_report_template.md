# 버그 리포트 작성 템플릿

## 사용법
사용자가 버그 내용을 입력하면 아래 형식으로 작성.

---

## 입력 형식
사용자가 보내준 버그 내용 (자유 형식)

## 출력 형식

**[현상]**
(입력 내용에서 어떤 문제가 발생하는지 명확하게 정리)

**[재현 스텝]**
1. (입력 내용을 기반으로 재현 가능한 단계 순서대로)
2. ...
3. ...

**[기대 결과]**
(정상적으로 동작해야 하는 상태 기술)

**[연관 문서]** ← 링크가 있을 때만 포함
- 링크 제목: URL

---

## 작성 규칙
- 입력 내용에 없는 내용은 임의로 추가하지 말 것
- 재현 스텝은 QA/개발자가 직접 따라할 수 있도록 구체적으로
- 기대 결과는 "~해야 합니다" 형태로 마무리
- 연관 문서 링크가 없으면 해당 항목 생략
