# 바이블테이블링 ID 규칙

## 핵심 규칙
- PK/FK 컬럼에 `_new`, `_v2`, `_temp` 같은 **비정규 접미사를 절대 붙이지 말 것**
- ID는 반드시 해당 컬럼의 자료형(보통 int32)에 맞는 **순수 정수값**이어야 함
- 기존 ID와 충돌할 경우, 같은 자릿수 범위 내에서 **다음 순번 정수**를 사용

## 예시
- ❌ `10013011_new` — 문자열 접미사, int32 불가
- ❌ `1001301-2` — 하이픈 포함, int32 불가
- ✅ `10013013` — 기존 10013011, 10013012 다음 순번
- ✅ `10013014` — 순차 증가

## 충돌 방지 워크플로우
1. 새 ID가 필요할 때 → 기존 데이터에서 해당 테이블의 ID 패턴 확인
2. 같은 그룹(prefix) 내에서 마지막 순번 확인
3. 마지막 순번 + 1로 새 ID 할당
4. query_game_data로 해당 ID가 미사용인지 반드시 검증

## 발생 배경
프리드웬 궁극기 리뉴얼 시 SkillExecuteEffect ID를 `10013011_new`로 지정하여 문제 발생.
int32 PK에 문자열이 들어가면 FK 참조 실패 및 데이터 로드 에러 발생.
