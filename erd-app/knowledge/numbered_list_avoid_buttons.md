# 번호 매기기 목록 — 객관식 버튼 충돌 방지 규칙

## 문제
채팅 UI에서 `1.` `2.` `3.` 또는 `A)` `B)` `C)` 형식의 번호 목록이 **객관식 버튼으로 자동 변환**되어 버림.
→ 분석 결과, 개선 방향, 나열형 설명 등에서 의도치 않게 클릭 버튼이 생성됨.

## 규칙
- **객관식 질문이 아닌 일반 나열/설명**에서는 번호 매기기(`1.` `2.` `A)` `B)`) 사용 금지
- 대신 아래 대체 표현 사용:

### ✅ 허용되는 대체 표현
1. **불릿 포인트** 사용: `- `, `• `, `▸ `
2. **굵은 키워드 + 설명** 형태:
   - **자유 형태 거점** — 맵마다 자유로운 거점 형태 사용 가능하도록 변경
   - **FX 상태 교체** — 머티리얼뿐 아니라 FX도 상태별로 교체 가능하도록 기능 추가
3. **표(테이블)** 형태로 정리

### ⛔ 금지 패턴 (객관식 버튼으로 오인됨)
```
1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목
```
```
A) 첫 번째
B) 두 번째
```

### ✅ 올바른 패턴
```
- **자유 형태 거점** — 맵마다 자유로운 거점 형태 사용 가능
- **FX 상태 교체** — 상태별 FX 교체 기능 추가
- **이질감 해소** — 맵 테마에 맞는 거점 디자인 적용
```

## ⭐ 객관식 선택지 간격 규칙 (2026-06 추가)
- 선택지를 버튼으로 표시하려면 **선택지 사이에 빈 줄(엔터) 넣지 말고 붙여쓸 것**
- 빈 줄이 들어가면 버튼으로 변환되지 않음

### ✅ 올바른 객관식 (붙여쓰기 → 버튼 변환됨)
```
어떤 항목을 원하시나요?
A) 첫 번째 선택지
B) 두 번째 선택지
C) 세 번째 선택지
```

### ⛔ 잘못된 객관식 (빈 줄 → 버튼 변환 안 됨)
```
어떤 항목을 원하시나요?

A) 첫 번째 선택지

B) 두 번째 선택지

C) 세 번째 선택지
```

## 적용 범위
- 채팅 본문의 모든 나열형 텍스트
- 분석 결과, 개선 방향, 요약, 비교 등
- 아티팩트 내부 HTML은 해당 없음 (아티팩트는 버튼 변환 안 됨)
