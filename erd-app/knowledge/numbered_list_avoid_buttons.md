# 번호 매기기 목록 — 객관식 버튼 충돌 방지 규칙

## 문제
- UI가 숫자로 시작하는 목록 항목을 객관식 버튼으로 변환함
- 예: `1) 텍스트`, `1. 텍스트` → 버튼으로 렌더링되어 클릭 가능 상태가 됨
- 사용자가 버튼을 클릭하면 의도치 않은 선택이 발생

## 규칙
- ❌ `1) ...`, `2) ...` — 숫자+괄호 목록 사용 금지
- ❌ `1. ...`, `2. ...` — 숫자+점 목록도 UI에 따라 버튼 변환될 수 있음
- ✅ **객관식 질문은 반드시 알파벳(A/B/C)으로!**
  - `A) ...`, `B) ...`, `C) ...`
  - `A-1)`, `A-2)` 처럼 세부 항목도 알파벳 기반으로
- ✅ 일반 목록은 `-` 또는 `*` 사용

## 적용 대상
- 사용자에게 선택지를 제시하는 모든 객관식 질문
- 복수 선택 체크리스트: `- [ ]` 형식 사용 (숫자 없이)

## 예시 (올바른 형식)
```
A) 기존 데이터 복사
B) 신규 데이터 생성
C) 직접 입력

A-1) CharacterStat만
A-2) 전체 연관 테이블
```
