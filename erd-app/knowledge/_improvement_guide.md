## ❌ 개선해야 할 패턴 (1건)

아래 유형의 질문에서 사용자가 불만족했으므로 **다른 접근법**을 사용하세요:

1. ❌ 결과 = 각 거점을 보고 싶어!!! 3개 모두! — 평가: 😕 도움이 안 됐어요 → "요청에 대해 행동을 하지 않았어요" [널리지: read:test_case_writing_rules, read:workflow_data_qa, read:wo]

## 📋 개선 방향

사용자가 가장 많이 선택한 불만족 원인: **도움이 안 됐어요** (1회)
→ 이 부분을 최우선으로 개선하세요.

- 긍정 패턴과 유사한 질문 → 동일한 도구와 답변 형식 사용
- 부정 패턴과 유사한 질문 → 이전과 다른 접근법 시도 (다른 쿼리, 더 상세한 설명 등)
- 사용자 의견이 명시된 경우 해당 의견을 최우선으로 반영
- 불확실한 정보는 "확인 필요"로 표시하고, 쿼리 오류 시 대안 제시
