# Jira 댓글 작성 스타일 규칙

## 1. 서식 규칙
- Jira 댓글 작성 시 마크다운 서식을 사용하지 않는다.
- ##, ###, **굵게**, ---, - 리스트 등 마크다운 기호 금지.
- Jira 웹 UI에서 마크다운이 렌더링되지 않고 원문 그대로 노출되기 때문.

## 2. 올바른 서식 스타일
- 순수 평문(plain text)으로 작성
- 구조화가 필요하면 줄바꿈과 들여쓰기, 번호 매기기(1. 2. 3.)로 구분
- 강조가 필요하면 대문자나 괄호, 화살표(→) 등 텍스트 기호 활용
- 코드나 변수명은 백틱 없이 그냥 텍스트로 표기

## 3. 어휘/톤 규칙 (주니어 개발자 대상)
- 주니어 레벨의 팀원이 읽는다고 가정하고 작성한다.
- 전문 용어를 쓸 때는 반드시 짧은 부연 설명을 괄호로 붙인다.
  예: 클라이언트 예측(클라가 서버 응답 안 기다리고 먼저 처리하는 것)
- 한 문장은 짧게. 한 문장에 하나의 의미만 담는다.
- "~라고 추정됩니다", "~일 가능성이 있습니다" 같은 장황한 표현 대신
  "~인 것 같습니다", "~때문입니다" 처럼 간결하게 쓴다.
- 기술적 인과관계는 "A → B → C" 화살표 흐름으로 보여준다.
- 비유나 일상 표현을 적극 활용한다.
  예: "예약만 해놓고 취소를 안 한 상태", "장부에는 남아있는데 실물은 없는 상황"
- 핵심 결론/의심 지점은 맨 위에 먼저 쓰고, 상세 설명은 아래에 붙인다.
- 불필요한 수식어, 반복 설명을 제거한다. 같은 말을 두 번 하지 않는다.

## 4. 구조 규칙
- [한줄 요약]을 맨 위에 둔다.
- 시간순/단계별로 번호를 매겨 흐름을 보여준다. (1단계 → 2단계 → 3단계)
- 의심 지점(수정 포인트)은 마지막에 명확하게 한 줄로 정리한다.

## 5. 예시
```
[한줄 요약] 재장전 취소할 때 예약 탄약 수를 안 지워서 음수 탄약이 나옵니다.

좀 더 풀어서 설명하면:

1단계: 탄약 0발 → 재장전 시작
탄약이 다 떨어지면 자동으로 재장전이 시작됩니다.
이때 "몇 발 채울 건지" 미리 예약해둡니다. (이게 _reservedAmmoCount)

2단계: 재장전 도중에 점프+사격 → 재장전 취소
재장전 중간에 다른 행동을 하면 CancelReload()가 불립니다.
문제는 여기서 _reservedAmmoCount를 0으로 안 돌려놓는다는 겁니다.

3단계: 다시 사격 시도
"실제 탄약 - 예약된 탄약"으로 남은 탄약을 계산하는데,
예약값이 안 지워져서 0 - 4 = -4 이런 식으로 음수가 나옵니다.

[의심 지점] CancelReload()에 _reservedAmmoCount = 0 한 줄이 빠져있습니다.
다른 초기화 함수(OnDeath, OnReset 등)에는 다 있는데 여기만 없습니다.
```
