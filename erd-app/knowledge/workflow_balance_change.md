# 밸런스 조정 워크플로

## 트리거
"밸런스", "수치 조정", "HP 올려", "ATK 변경", "너프", "버프", "상향", "하향", "스탯 변경", "배율 조정" 등

## 워크플로 단계

### 1단계: 영향 범위 분석 (편집 전 반드시!)
- 대상 테이블의 FK 참조를 **역방향**으로 추적
- query_game_data로 관련 테이블에 해당 데이터가 얼마나 있는지 카운트
- :::visualizer{type="relation"} 으로 영향 관계도 시각화
- 사용자에게 "이 테이블들도 함께 조정할까요?" 형태로 확인

#### 예시: "몬스터 HP 10% 상향"
```
영향 범위:
- Monster (직접 대상) — 50행 해당
- MonsterDrop (Monster 참조) — 관련 120행
- StageMonster (Monster 참조) — 관련 80행
→ "Monster HP만 조정할까요, 아니면 드랍/보상도 함께 조정할까요?"
```

### 2단계: 변경 전 스냅샷
- 수정 대상의 **현재 값**을 query_game_data로 조회하여 기록
- 대표 행 5~10개를 마크다운 표로 보여주기
- 이 스냅샷은 4단계 비교에 사용

### 3단계: 편집 실행
- 사용자가 승인한 범위만 edit_game_data 실행
- 비율 변경 시 `multiply` action 사용 (set으로 하나하나 계산하지 말 것)
- 절대값 변경 시 `add`/`subtract` action 사용
- 대량 편집 시 csv_set 사용

### 4단계: 변경 전후 비교
- 2단계 스냅샷과 동일 데이터를 다시 조회
- :::visualizer{type="diff" title="밸런스 변경 전후"} 로 비교 시각화
- 변경률이 극단적이면 경고:
  - 100% 이상 증가 또는 50% 이상 감소 → "변경폭이 큽니다. 의도한 것인지 확인해주세요."
  - 값이 0이 되는 경우 → "이 값이 0이 되면 게임에서 문제가 생길 수 있습니다."

### 5단계: 검증 + 요약
- validation rule이 등록되어 있으면 검증 실행
- 범위 룰 위반 확인 (min/max)
- 변경 요약을 텍스트로 정리:
  - 대상 테이블, 영향 행 수, 변경 내역
  - 주요 수치의 이전/이후 대표값

## 주의사항
- 밸런스 조정은 **한 테이블만 바꾸면 다른 테이블과 불균형**이 생길 수 있음
- 항상 "관련 데이터도 확인할까요?" 를 물어볼 것
- 변경 전 스냅샷을 꼭 남겨서, 문제 시 원래 값 참조 가능하게
- 변경 후 즉시 검증 — 위반 없으면 "검증 통과" 명시
